1. Objectif produit

Passer d’une simple liste commentée à un mini-rapport : synthèse chiffrée, graphiques, tableau, exports — dans le même esprit visuel que le reste du chat MONV.

Promesse : vue d’ensemble lisible + tableau exploitable + boutons d’export déjà familiers (Excel blanc, CSV discret), sans conseil en investissement ni juridique.

2. Périmètre

Inclus

  • Critères clairs (activité, zone, taille) à partir de la question.
  • Vue marché avec effectifs statistiques (X entreprises sur Y) par indicateur.
  • Tableau + filtres ; export = même contenu filtré.
  • Bloc sources et date.

Exclu

  • Pas de recommandation d’achat/vente, pas de valorisation, pas de conseil personnalisé.
  • Pas de mélange visuel entre chiffres « liste MONV » et statistiques nationales externes sans encart séparé.

4. Exemple : cabinets d’expertise comptable

Exemple de question : effectifs 10–200, France, chiffre d’affaires et évolution. Le titre du dossier et les filtres doivent refléter ce périmètre ; le badge de mode reste Benchmark (style ambre comme dans l’app).

5. Parcours

  1. Choix du mode Benchmark + saisie.
  2. Éventuellement QCM si la question est trop large.
  3. Résultat : écran dossier (onglets Panorama → Insights) ou emboîtement progressif depuis le message « résultats » actuel.
  4. Filtres → recalcul des agrégés sur la liste visible.
  5. Export pack ou Excel/CSV comme aujourd’hui.

Annuaire entreprises, enrichissements financiers selon abonnement ; l’onglet Sources liste le détail pour le dossier courant.

9. Exports

  • Résumé (mêmes KPI que l’écran + complétude).
  • Détail (lignes = tableau filtré).
  • Sources + date.

10. Règles UI

  • Réutiliser les composants / classes du chat : carte rounded-xl border-border bg-card.
  • Mode Benchmark : accent ambre (badges, bordures actives), comme MODE_META.benchmark.
  • Bouton export principal : dans l’app en thème sombre, fond blanc et texte très foncé ; sur support papier / PDF noir sur blanc, un bouton noir avec texte blanc reste lisible et équivalent visuellement.
  • Texte secondaire : text-muted-foreground.

11. Ordre de développement suggéré

  1. Squelette page : barre globale (exports, partage) + hero (badges, titre, ligne de fraîcheur des sources).
  2. Filtres : barre type chips (effectif, CA, région, etc.) branchée sur le même état que le tableau.
  3. Rangée KPI principale (6 tuiles) + recalcul quand les filtres changent.
  4. Onglets : Panorama puis Finance, Dynamique, Carto, Classements, Tableau, Insights — au minimum coquilles vides + contenu Panorama + Tableau existant.
  5. Graphiques : histogrammes / donut / radar avec données réelles ou mock ; export image optionnel.
  6. Insights IA : liste d’extraits sourcés ; encart « Sources » aligné sur le mock v2.
  7. Pack export : XLSX multi-feuilles + PDF long format quand le produit valide.
